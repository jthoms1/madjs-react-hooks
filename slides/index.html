    - Hooks are a new feature proposal that lets you write only functional components and use React features without needing a class. - First announced at React Conf last month and since then it has really blown up in popularity. - I was at React Conf and that's all anyone could talk about. - Vue is dabbling with implementing. - A hook is a special function that lets you hook into React features

    Why is React changing again

    • Stateful Logic is hard to reuse today
    • Lifecycle makes components sometimes hard to reason about
    • Classes are not the ideal solution
    I say again because it first it was createClass and mixins. - Stateful Logic is hard to reuse - Render Props and Higher-Order Components - Patterns can be cumbersome and make you restructure your app's tree to maintain them - React realized the primitives needed to change for better sharing (rare for them) - Lifecycle makes components sometimes hard to reason about - Hate refactoring functional components to Class components because you need Lifecycle events or to add state - Ever had a simple component get a little too big? - then you split it up into more components for reuse and then we get back to stateful logic is hard to reuse - Classes are not the ideal solution - Issues with minification, hot reloading, they don't treeshake

    How will this affect my codebase

    • Meant to be a gradual enhancement
    • Work side by side with class based components
    • Get a team consensus on usage
    - This is an opt in feature. Classes are not planned to be removed - They work side by side with all existing features - Obviously you will need to work with your team to deside if it is time to start the transition. - It might be very attractive to do so, but remember the code will look quite different

    Hooks API

    • useState
    • useEffect

    useState

    - function returns an array - [0] is the current value, [1] is the setter - Easy to understand

    useEffect

    - side effects in a function - componentDidMount, componentDidUpdate, componentWillMount - set a function to be executed after render is done - gets rid of some duplicates with DidMount, DidUpdate - runs after every render - does not block - return a function if you want it to be executed for cleanup. Think WillUnmount - fantastic for subscribe, unsubscribe - accepts second parameter as array of values to check if a value is different to rerun

    • Hooks Rules

    Don’t call Hooks inside loops, conditions, or nested functions. Don’t call Hooks from regular JavaScript functions
    - Hooks need to ALWAYS preserve the order that they are called.
